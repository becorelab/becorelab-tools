## 1. 한 줄 요약

두리(Claude Desktop)가 MCP 도구 60개+를 한꺼번에 로드하느라 세션 토큰을 너무 빨리 소진해서,
카테고리별로 **4개 서버로 분리**했어요. 이제 필요한 것만 켜면 **토큰 최대 80% 절약** 가능합니다.

---

## 2. 왜 했는지 (배경)

- 두리(사무실 Claude Desktop)가 전체 MCP 도구 61개를 로드하면 세션 시작 시점에 이미 **12,000~18,000 토큰** 차지
- 긴 대화를 못 이어가고 한도가 금방 소진되는 문제 발생
- 해결책: 작업 종류별로 서버를 나누고, 평소엔 `sales`만 켜두고 필요할 때만 추가 활성화
- 기존 `server.py`(전체 통합)는 그대로 유지 — 원하면 예전 방식도 사용 가능

---

## 3. 효과 (대표님 보고용)

| 활성화 구성 | 예상 토큰 | 절감률 |
|---|---|---|
| 현재 (server.py 전체 61개) | 약 12,000~18,000 | 기준 |
| `sales`만 | 약 2,500 | **80% 절감** |
| `sales` + `memo` | 약 4,000 | **70% 절감** |
| `sales` + `ads` | 약 5,500 | **60% 절감** |
| `sales` + `sourcing` | 약 9,500 | **35% 절감** |
| 4개 전부 | 약 13,000 | 기존과 비슷 |

**결론**: 평소엔 `sales`만 켜고, 작업 종류 바뀔 때만 추가 활성화하면
세션 한도를 **5배 가까이 길게** 사용할 수 있습니다.

## 7. 사용 시나리오 가이드

### 시나리오 A: 아침 매출 확인
- ON: `sales`만
- 토큰: ~2,500
- 할 일: 전날 매출 / 주문 현황 / 물류 상태 조회

### 시나리오 B: 메타 광고 최적화
- ON: `sales` + `ads`
- 토큰: ~5,500
- 할 일: 매출 확인 후 광고 성과 대조, 예산 조정

### 시나리오 C: 신규 상품 소싱
- ON: `sales` + `sourcing`
- 토큰: ~9,500
- 할 일: 알리바바 검색, 골드박스 등록, 원가 계산

### 시나리오 D: 회고 / 문서 정리
- ON: `sales` + `memo`
- 토큰: ~4,000
- 할 일: 옵시디언 메모 작성, 이미지 생성, 주간 리포트

### 시나리오 E: 전체 작업 (풀 모드)
- ON: 4개 전부 or `server.py` 1개
- 토큰: ~13,000~18,000
- 할 일: 복합 작업, 기존과 동일

## 8. 롤백 방법

혹시 문제가 생기면 기존 방식으로 즉시 복귀 가능합니다.

```json
{
  "mcpServers": {
    "becorelab": {
      "command": ".venv/bin/python",
      "args": ["server.py"],
      "cwd": "/Users/kymac/claude/mcp-server"
    }
  }
}
```

`server.py`와 `server_remote.py`는 건드리지 않았기 때문에 언제든 원상복귀 가능합니다.

## 9. 주의사항

- **도구 이름 충돌 없음** — 각 서버가 독립 프로세스로 뜨기 때문에 이름이 겹쳐도 괜찮음
- **하지만 같은 카테고리 도구를 두 서버에 동시 등록하면 혼란 발생** → 카테고리 규칙 지키기
- **Claude Desktop 재시작 필수** — 설정 바꾸면 앱 완전 종료 후 재실행
- **가상환경 활성화 확인** — `.venv`에 `mcp`, `fastmcp`, `httpx` 설치되어 있어야 함
- **포트/소켓 사용 안 함** — stdio 방식이라 포트 충돌 걱정 없음
