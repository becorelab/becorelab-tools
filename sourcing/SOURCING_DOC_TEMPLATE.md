## 📐 문서 구조 = 2층

### 상단 = 소싱 기술서 (상아님 실행용) — 6섹션
상아님이 이것만 보고 1688·알리·도매꾹에서 소싱 실행. 간결·실행 중심.
1. **무엇을 찾는가** — 제품 정의·용도
2. **시장 현황** — 기회점수 · TOP10 평균 월매출 · 평균 판매가 · 가격대 3층 · 1위 상품
3. **상위 10개 경쟁 상품** — 소싱박스 추적 매출 1~10위 테이블(순위·상품·판매가·월매출·리뷰·**클릭 링크** `[보기](쿠팡url)`). 시장 현황 바로 뒤(상아님 파트 상단)
4. **소비자 인사이트** — 소구점 표(순위·니즈·건수·비중) + 페인포인트 표 + `> 💡 핵심 빈자리`
4. **타겟 스펙** — 판매자에게 요구할 필수 사양(최우선 항목 표시)
5. **타겟 랜딩원가** — 표: 포지션·판매가·타겟랜딩원가·**1688 목표단가(¥)**·**알리바바 목표단가($)** + 산식 주석
6. **판매자 확인 질문** — 제품 특화 + 표준(MOQ·수량별단가·커스텀·샘플비·납기·인증)
   - 끝에 `> 📌 상아님 TIP` + "업체는 🏭 소싱 업체 후보 DB에 기록 👇"

### 하단 = 내부 문서 (우리 의사결정용) — 리뷰 심층 + 종합 전략 9섹션
> ⚠️ 어제 렌더 이슈: 표 셀 안 `**볼드**`는 노션에서 리터럴로 남음 → 표 셀엔 볼드 쓰지 말 것. 방대해서 토글보다 섹션이 안전(대표님 취향 확인).

**📊 리뷰 심층 분석 (N건 전수)** — 근거 데이터
- 별점 분포(긍정/불만 비율) · usage/needs/pain 전수 집계표
- 텍스트 바차트(인라인코드 `████` + `<br>`) · **실제 리뷰 원문**(만족 3 / 불만 3, 상품명 표기) · 색상/구성 분포 · 소비자 인사이트 한 줄

**📊 내부 종합 전략 (소싱 실행) — 9섹션**
1. **종류/세그먼트** — 하위 유형·특징·가격대·대표상품·진입매력도(★)
2. **소비자 선호(needs)** — 표 + JTBD 해석
3. **페인포인트** — 표 + 불만 원문 + 전략적 함의
4. **시장 가격대** — 밴드별·구성별(1개입/멀티팩)
5. **🏭 소싱처 분석** ← **핵심**
   - 1688 실검색 후보표(상품·단가¥·id·판단) — `search_1688.py "중국어키워드"`
   - **랜딩코스트 계산**(1688단가+중국물류+관세+검수+국내운송, 대량 vs 소량)
   - **마진 시뮬레이션**(판매가별 공헌이익%, 물류포함/미포함)
   - 샘플 주문 후보 3~5개 + 공급사 검증 체크리스트
6. **판매 전략** — 포지셔닝(JTBD 4힘)·채널·SKU 로드맵
7. **가격·옵션 구성** — 판매가 확정안·옵션·번들
8. **상세페이지 방향** — 핵심 소구(1위 페인 정면공략)·강조순서·과대광고 회피
9. **GO 결론** — GO / 조건부 / 신중 판정 + 근거 한 줄

## 🔄 작성 순서 (파이프라인)
1. **시장 스캔** — `sourcing_scan_and_wait("키워드")` → 기회점수·매출분포·상위상품
2. **DB 속성 계산** (매출기준): 메인키워드 / 300만↑셀러수(N/전체) / 상위3매출합(만원) / 상위3판매량합 / 4-10등매출평균(만원) / 4-10등판매량평균 / 상위3판매가평균(원) — 숫자 콤마 표기
3. **리뷰 전수분석** — `sourcing_reviews_start(scan_id)` 수집 → `review_pipeline.py extract` → 서브에이전트 전수태깅 → 집계 (REVIEW_ANALYSIS.md 방법론)
4. **1688 실검색** — `search_1688.py "중국어키워드"` (예: 运动发带/地漏防臭器/硅胶洗头刷)
5. **랜딩코스트·마진 시뮬** — 위 파라미터로 계산
6. **문서 작성** — 상단 기술서 + 하단 종합전략, 노션 상품기획 DB 제품 페이지로
7. **소싱 업체 후보 DB** 연결(relation) — 상아님이 채움
