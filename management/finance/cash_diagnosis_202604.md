## 3. 범인 ②: 올라 선정산 상계 — 월마다 널뛰는 현금

### 올라 선정산이란?

쿠팡 매출채권을 담보로 **미리 현금을 받는 회전대출**. 쿠팡 정산(58일)이 들어오면 올라가 먼저 상계(회수)한다.

### 핵심 발견 ①: 쿠팡 정산 = 올라 상계 (100%, 같은 날, 같은 금액)

하나은행 거래내역을 날짜별로 대조한 결과:

```
4/24: 쿠팡 입금 8,720,480원 → 올라 상계 8,720,480원 ✅ 100% 일치
4/21: 쿠팡 입금 15,764,400원 → 올라 상계 15,764,400원 ✅ 100% 일치
3/06: 쿠팡 입금 16,259,175원 → 올라 상계 16,259,175원 ✅ 100% 일치
2/13: 쿠팡 입금 10,251,386원 → 올라 상계 10,251,386원 ✅ 100% 일치
```

**쿠팡이 정산하는 날, 올라가 같은 금액을 100% 가져간다.**
쿠팡 정산금 중 회사가 직접 쓸 수 있는 돈 = **0원**.

실제 현금 흐름:
```
[쿠팡 매출] → 58일 뒤 정산 → [하나은행 입금] → [올라 100% 상계] → 회사에 남는 돈: 0원
[올라 선정산] → 회사 요청 시 advance → [하나은행] → [신한은행 이체] → 운영자금
```

**쿠팡 매출에서 실제로 쓸 수 있는 현금 = 올라 선정산(advance) 금액뿐.**
올라 선정산 = 빚을 늘리는 것 (월 2% = 연 24%).

## 7. 구조 진단 요약

### 비코어랩의 현금 구조

```
[사업]  매출 → 원가·광고·고정비 차감 → 흑자 (+223만) ✅
          │
          ├── [대출] 원리금 982만 고정 유출 ──────── 매월 확정 출혈
          │
          └── [올라] 쿠팡 정산 58일 → 올라 선정산으로 당겨받음
                     정산 도착 → 올라가 먼저 상계
                     상계가 입금보다 크면 → 추가 현금 유출
                     상계 NET: -601만 ~ -2,133만 (매월 변동)
```

### 왜 항상 돈이 없는가 — 3줄 요약

1. **사업은 돈을 벌고 있다** — 대출 빼면 월 +223만 흑자
2. **대출 원리금(982만)이 흑자를 잡아먹는다** — 특히 부장님 500만이 큼
3. **올라 상계가 월마다 -600만~-2,100만씩 추가로 빠진다** — 이게 "3천만원"의 변동 요인

| 3 | 대출 원리금이 흑자를 적자로 뒤집음 | 982만/월 고정 유출 |

| 5 | 올라 수수료 = 연 24% | 월 ~160만 (사채 수준) |

### 구조를 바꾸는 3가지 레버

경비 절감은 이미 대표님이 하셨고 효과도 냈다. 남은 부족분은 **대출 구조**와 **올라 구조**를 건드려야 줄어든다.

**레버 ① 대출 구조 개선** (최우선)
- 982만/월 → 줄일 수 있는 부분 식별
- 부장님 500만: 상환 조건 재협상 가능성 검토 (거치기간, 감액 등)
- 고금리 대출(IM캐피털 89만, IBK사잇돌 116만) → 저금리 대환 검토
- 원리금이 500만 이하로 내려가면 사업 흑자만으로 커버 가능

**레버 ② 올라 의존도 축소** (중기)
- 연 24% 수수료는 사채 수준 — 쓸수록 현금이 샌다
- 올라 사용액 8,185만 → 단계적 축소 (선정산 요청 금액 줄이기)
- 대안: 시중은행 매출채권 담보대출 (연 5~8%) 또는 쿠팡 자체 빠른정산
- 올라 상계 NET이 0에 가까워지면 → 월 부족분이 대출 원리금 수준으로 축소

**레버 ③ P&L 흑자 폭 확대** (기본)
- 현재 대출 제외 +223만 → +500만 이상으로 키우기
- 쿠팡 광고비(1,579만) 추가 효율화 여지 검토
- 비쿠팡 채널(스마트스토어, 자사몰) 매출 비중 확대 → 정산 빠름 + 수수료 낮음

### 우선순위 (하치의 소견)

```
1순위: 대출 구조 개선 (레버①)
  → 982만 → 500만 이하로 줄이면 사업 흑자로 커버 가능
  → 부장님 상환 조건 재협상이 가장 큰 임팩트

2순위: 올라 의존도 축소 (레버②)
  → 연 24% 수수료 탈출 + 상계 변동폭 축소
  → 레버①과 동시 추진 가능

3순위: P&L 흑자 폭 확대 (레버③)
  → 광고 효율화 + 비쿠팡 채널 확대
  → 1·2순위와 시너지
```

### 목표 시나리오

```
현재:  +223 - 982 - (601~2,133) - 160 = -1,520 ~ -3,052만
                                          ↓
대출 500으로:  +223 - 500 - (601~2,133) - 160 = -1,038 ~ -2,570만
                                          ↓
올라 NET 0으로: +223 - 500 - 0 - 80 = -357만
                                          ↓
흑자 500으로:   +500 - 500 - 0 - 0 = ±0 (손익분기)
```

**대출 구조 개선 + 올라 탈출 + 흑자 확대 → 가수금 투입 0원이 가능한 구조.**

## 9. 결론

**대표님이 지금까지 하신 건 정확히 맞았다.** 인원 감축과 광고 효율화로 사업 자체를 흑자로 만들었다.

통장이 안 나아지는 이유는 사업 문제가 아니라 **대출 원리금(982만)** + **올라 상계 타이밍 변동**이라는 별개의 구조 문제다. 고정비를 더 줄여도 이 두 가지는 해결되지 않는다.

핵심 공식 (하나은행 7개월 검증):
```
최고의 달(25/11): +223 - 982 - 160 + 1,419 = +500만 (유일한 흑자)
평균:             +223 - 982 - 160 - 771   = -1,690만
최악의 달(25/12): +223 - 982 - 160 - 2,936 = -3,855만
```

**"매달 3천만원 부족"은 매달이 아니다. 올라 상계가 몰리는 달(12월 -3,855만, 4월 -3,052만)에 집중된다.**
**반대로 올라 NET이 플러스인 달(11월 +500만, 2월 -293만)은 거의 손익분기에 근접한다.**
