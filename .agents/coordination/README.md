# 비코어랩 하치 협업·충돌 방지 규칙

이 폴더는 Claude 하치와 GPT 하치가 같은 저장소에서 병행 작업할 때 사용하는 모델 중립 협업 장부다.

## 원칙

1. **메인 역할과 파일 소유권은 다르다.** GPT 하치는 전체 맥락·배분·최종 검증을 맡지만, 모든 파일을 독점하지 않는다.
2. **영구적인 모델별 코드 소유권을 두지 않는다.** 작업마다 담당자와 수정 범위를 `ACTIVE_TASKS.md`에 등록한다.
3. **한 파일은 한 시점에 한 담당자만 수정한다.** 다른 담당자의 `in_progress` 범위와 겹치면 읽기·분석까지만 하고 수정하지 않는다.
4. 작업 전 `git status --short`와 대상 파일의 `git diff`를 확인한다. 소유자를 모르는 변경은 보존한다.
5. DB·정산 원장·구글시트·외부 관리자 화면은 작업별 **단일 작성자**만 반영한다. DB·원장 쓰기 전에는 백업·영향 범위·복원 방법을 확인한다.
6. 장기·대규모 코드 작업이 겹치면 별도 Git worktree/브랜치를 사용한다. 커밋·푸시는 대표님 지시가 있을 때만 한다.
7. 완료·중단 시 장부 상태를 갱신하고, 남은 변경·검증·주의사항을 인수인계에 남긴다.

## 장부 작성 형식

| 시작시각 | 담당 | 작업 | 수정 범위 | 상태 | 비고 |
|---|---|---|---|---|---|
| KST ISO 시각 | GPT 하치 / Claude 하치 | 짧은 작업명 | 정확한 파일·디렉토리 | `in_progress` / `done` / `blocked` | DB·외부 반영 여부 |

`수정 범위`는 `erp/**`처럼 지나치게 넓게 잡지 말고 실제로 수정할 파일까지 좁힌다.
