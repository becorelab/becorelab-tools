## 6. JSON 구조 (계산 에이전트용)

```json
{
  "meta": { "raw_name_column": "...", "raw_files": {...}, ... },
  "name_map": { "raw 품명": "standard", ... },      // 632개
  "cost_map": { "standard": cost_vat_incl, ... },   // 131개
  "cost_map_monthly": { "2025-01": {...}, ... },    // 월별 원가 fallback
  "raw_appearance": { "raw 품명": ["2025-01", ...] },
  "stats": {...},
  "issues": {...}
}
```

### 사용 규칙
1. raw 품명을 찾을 때 `name_map[raw]` 직접 조회 (문자열은 **strip만** 된 상태).
2. 조회 안 되면 `re.sub(r'\s+', '', raw)` 후 재시도할 수 있게, 필요시 호출부에서 한번 더 정규화 매핑을 만들어도 OK.
3. 원가는 `cost_map[standard]` 우선. 없으면 `cost_map_monthly[month][standard]` fallback.
4. 원가 단위: **판관비포함원가 (VAT 포함)** — 통합 원가파일의 E열.
5. 복합 구성 standard는 원가가 없을 수 있음 — 계산 에이전트는 이 케이스를 별도로 리포트할 것.
