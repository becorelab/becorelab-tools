- **시트**: `원본` (헤더 1행)

## 계산 로직
1. 모든 행을 순회하며 `주문 상태` 필터링 (배송 완료 / 구매 확정 / 배송중 / 배송 준비중)
2. `gross = 수량 × 옵션+판매가` 를 행마다 누적
3. 차감 항목은 **별도 누적 후 마지막에 일괄 차감**
   - 주문번호 기준 1회만 적용: `쿠폰 할인금액(최초)`, `사용한 적립금액(최종)`, `총 배송비(KRW)`
   - 행마다 누적: `실제 환불금액`, `회원등급 추가할인금액`, `앱 상품할인 금액(최종)`, `상품별 추가할인금액`
4. 원가: `주문상품명(옵션포함)` → `name_map` → `standard` → `cost_map` (또는 `cost_map_monthly[월]` fallback) → 단가 × 수량
5. 최종:
   - `total_discount = coupon + refund + point + rank + app + extra`
   - `net_revenue = gross - total_discount`
   - `profit = net_revenue - cost`
   - `margin_rate = profit / net_revenue`

- 만약 모리와 수치 차이가 발생한다면 원인 후보:
  1. 주문번호 dedup 로직의 순서 차이 (첫 등장 기준 vs 마지막 등장 기준)

- 픽시(정산시트 역추적)와의 차이는 실제 정산시트의 수동 보정 여부에 달려 있음.
